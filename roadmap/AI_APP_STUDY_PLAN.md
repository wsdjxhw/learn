# AI 应用开发压缩学习计划

目标：尽快具备独立开发 AI 应用的能力，而不是把语言、框架、算法同时学深。

原则：
- `Python` 作为主线，优先学到能写 AI 应用。
- `Go` 作为副线，学到能写简单后端服务即可。
- 每周都要有可运行代码，不做只看不练的学习。
- 先学最小闭环，再补框架和工程化。

## 总周期

- 第 1-2 周：压缩完成 `Python + Go` 语言入门
- 第 3-4 周：Python Web 开发与 AI API 接入
- 第 5-6 周：数据库、任务流、项目结构
- 第 7-8 周：RAG 和文档问答
- 第 9-10 周：AI 应用工程化与部署

## 第 1-2 周：语言压缩入门

目标：达到“能看懂示例、能改代码、能自己写小工具”的水平。

### 第 1 周：Python 最小必修

学习内容：
- 变量、字符串、列表、字典
- `if`、`for`、函数
- 文件读写
- JSON 处理
- HTTP 请求
- 虚拟环境和依赖安装

必须完成：
- 看懂 `examples/python/01_text_tool.py`
- 看懂 `examples/python/02_http_json.py`
- 跑通 `examples/python/03_fastapi_app`
- 自己改 2 次逻辑，例如增加一个字段、改返回格式、增加一个接口

输出目标：
- 能写命令行脚本
- 能调 HTTP API
- 能写一个最小 Web 接口

### 第 2 周：Go 最小必修

学习内容：
- 变量、切片、map、结构体
- 函数、方法、错误处理
- `goroutine` 和 `channel` 的基础概念
- `net/http`

必须完成：
- 看懂 `examples/go/01_struct_and_slice.go`
- 看懂 `examples/go/02_http_server.go`
- 自己增加一个新的 API 路由
- 自己把返回结构改成统一 JSON 格式

输出目标：
- 知道 Go 的基本写法
- 能写最小 HTTP 服务
- 理解 Go 在 AI 系统里更适合做接口层、网关、并发任务服务

## 第 3-4 周：进入 AI 应用开发

目标：把“会写代码”转换成“会写 AI 应用”。

### 第 3 周：Python Web + AI API

学习内容：
- `FastAPI` 路由、请求体、响应体
- 环境变量管理
- 调用大模型 API 的基本模式
- 对话消息结构
- 错误处理和日志

必须完成：
- 跑通 `examples/ai/01_chat_api`
- 理解接口为什么拆成 `provider.py` 和 `main.py`
- 给聊天接口增加一个参数，例如 `temperature` 或 `user_id`

输出目标：
- 能写一个最小聊天接口
- 理解 AI 应用的后端结构

### 第 4 周：做第一个完整小项目

项目建议：`AI 文本助手`

功能范围：
- 输入一段文本
- 选择任务类型：总结、改写、提取要点
- 返回结果
- 保存请求日志到本地文件

这一周不要追求前端复杂度，命令行或最简单的 HTML 页面都可以。

## 第 5-6 周：后端基础补齐

目标：让应用不是“一次性脚本”，而是像真正的小产品。

学习内容：
- `SQLite` 或 `PostgreSQL` 基础
- ORM 基础概念
- 用户、会话、消息表设计
- 后台任务基础概念
- 配置文件、分层目录、异常处理

项目建议：`带历史记录的聊天助手`

功能范围：
- 保存用户会话
- 查看历史消息
- 记录错误日志
- 支持简单的提示词模板

## 第 7-8 周：RAG 与知识库问答

目标：进入最常见的 AI 应用场景。

学习内容：
- 文本切分
- embedding 概念
- 向量检索
- 检索后再生成回答
- 来源引用

项目建议：`文档问答系统`

功能范围：
- 上传文档
- 切分并存储
- 根据问题检索片段
- 结合检索结果生成回答
- 返回引用来源

## 第 9-10 周：工程化与部署

目标：把 Demo 提升到能部署、能维护。

学习内容：
- Docker 基础
- `.env` 配置
- 日志
- 接口限流基础
- 简单鉴权
- 部署到云服务器或平台

项目建议：`可部署的 AI 助手 API`

功能范围：
- 健康检查
- 统一错误响应
- 接口密钥校验
- 容器化运行

## 每天怎么学

建议每天 `2-3 小时`，固定节奏：

- 30 分钟：看概念和官方文档
- 90 分钟：敲示例和改代码
- 30 分钟：复盘，记录今天学会了什么

## 学习顺序上的硬性建议

先学这些：
- Python 基础
- FastAPI
- HTTP/JSON
- 数据库基础
- AI API 调用
- RAG

晚一点再学这些：
- 复杂 Agent 框架
- 微服务拆分
- 高级前端框架
- 深度学习训练
- Kubernetes

## 你现在最该做的事

按这个顺序开始：

1. 先跑 `examples/python/01_text_tool.py`
2. 再跑 `examples/python/02_http_json.py`
3. 再启动 `examples/python/03_fastapi_app`
4. 再看 `examples/go/01_struct_and_slice.go`
5. 再启动 `examples/go/02_http_server.go`
6. 最后进入 `examples/ai/01_chat_api`

## 验收标准

学完前 2 周后，你至少要能独立完成下面 4 件事：

- 写一个 Python 命令行脚本
- 写一个 Python Web API
- 写一个 Go Web API
- 看懂一个最小 AI 聊天接口的结构

如果这 4 件事做不到，就不要急着继续学更重的 AI 框架。
