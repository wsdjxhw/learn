# AI 应用开发大方向学习计划

这份计划是项目的长期路线图。它不是要求一次性完成，而是告诉你每个阶段要解决什么问题。

## 阶段 1：编程语言最小入门

目标：能读懂示例代码，并能做小范围修改。

### Python

重点内容：

- 变量、字符串、列表、字典
- `if`、`for`、函数
- 文件读写
- JSON 处理
- HTTP 请求
- 虚拟环境和依赖安装

最低要求：

- 能运行 Python 脚本
- 能定义函数
- 能返回字典
- 能读取文件
- 能调用一个 HTTP API

对应示例：

- `examples/python/01_text_tool.py`
- `examples/python/02_http_json.py`

### Go

重点内容：

- 基础语法
- 结构体
- 切片
- map
- 错误处理
- `net/http`

最低要求：

- 能看懂 Go 的结构体和函数
- 能写一个最小 HTTP 服务
- 知道 Go 更适合做后端服务、网关、并发任务

对应示例：

- `examples/go/01_struct_and_slice.go`
- `examples/go/02_http_server.go`

## 阶段 2：Web API 基础

目标：理解“代码如何变成接口”。

重点内容：

- FastAPI 应用对象
- 路由
- 请求体
- 响应 JSON
- 路径参数
- 查询参数
- 常见 HTTP 方法

需要掌握的接口类型：

- `GET`：查询数据
- `POST`：新增数据
- `PUT`：整体替换数据
- `PATCH`：局部修改数据
- `DELETE`：删除数据

最低要求：

- 能启动 `uvicorn`
- 能打开 `/docs`
- 能用 `/docs` 测试接口
- 能新增一个接口
- 能区分路径参数、查询参数、请求体

对应示例：

- `examples/python/03_fastapi_app/main.py`
- `examples/python/04_fastapi_routes/main.py`

## 阶段 3：AI 接口调用

目标：理解 AI 应用后端最小结构。

重点内容：

- `.env` 配置文件
- API Key
- mock 模式
- 模型 provider 层
- 系统提示词
- 用户消息
- 模型回复

最低要求：

- 能配置 DeepSeek key
- 能在没有 key 时使用 mock
- 能看懂 `main.py` 和 `provider.py` 的分工
- 能调用 `/chat`

对应示例：

- `examples/ai/01_chat_api/main.py`
- `examples/ai/01_chat_api/provider.py`

## 阶段 4：聊天历史和数据库

目标：让 AI 应用从一次性问答变成有状态应用。

重点内容：

- 会话表 `sessions`
- 消息表 `messages`
- `session_id` 关联
- 保存用户消息
- 保存 AI 回复
- 查询历史消息
- 把历史消息传给模型

当前阶段使用 SQLite。

为什么：

- 不需要安装数据库服务
- 适合本地学习
- 能先掌握表设计和读写流程

后续再迁移到 PostgreSQL。

最低要求：

- 能创建会话
- 能发送消息
- 能查出历史消息
- 能说清楚 user / assistant 两类消息
- 能看懂 `db.py` 的数据访问函数

对应示例：

- `examples/ai/02_chat_history/main.py`
- `examples/ai/02_chat_history/db.py`
- `examples/ai/02_chat_history/provider.py`

## 阶段 5：ORM 和 PostgreSQL

目标：把学习 Demo 逐步靠近真实项目。

重点内容：

- SQLAlchemy
- 数据模型
- 数据库连接串
- PostgreSQL
- 数据库迁移
- Alembic

要解决的问题：

- 直接写 SQL 会越来越难维护
- SQLite 不适合多人在线生产环境
- 正式项目需要迁移和版本管理

最低要求：

- 能把 SQLite 版本迁移成 SQLAlchemy 版本
- 能把数据库切到 PostgreSQL
- 能理解表结构迁移

## 阶段 6：RAG 文档问答

目标：让 AI 能基于你自己的资料回答问题。

重点内容：

- 文档上传
- 文本切分
- embedding
- 向量库
- 相似度检索
- 检索结果拼接到 prompt
- 回答时返回引用来源

要解决的问题：

- 模型不知道你的本地文档
- 直接把长文档塞进 prompt 成本高，也容易超上下文
- RAG 让模型先检索资料，再回答

最低要求：

- 能上传一个文本文件
- 能把文本切成片段
- 能按问题检索相关片段
- 能带着片段调用模型
- 能返回来源

## 阶段 7：任务队列和异步处理

目标：处理耗时任务。

重点内容：

- 后台任务
- 状态查询
- 文件处理任务
- 失败重试
- Redis 基础
- 简单队列

典型场景：

- 上传文档后需要慢慢解析
- 生成长报告需要等待
- 批量处理文件不能阻塞接口

最低要求：

- 能提交一个任务
- 能查询任务状态
- 能区分同步接口和异步任务

## 阶段 8：认证、限流和日志

目标：让应用具备基本安全性和可维护性。

重点内容：

- API Key 鉴权
- 用户登录的基本概念
- 请求日志
- 错误日志
- 接口限流
- 统一错误响应

要解决的问题：

- 不能让任何人无限调用你的模型接口
- 模型调用有成本
- 出错后需要能定位问题

最低要求：

- 能给接口加一个简单鉴权
- 能记录每次请求
- 能统一返回错误格式

## 阶段 9：前端和产品化

目标：让后端能力变成可使用的应用。

重点内容：

- 简单 HTML 页面
- Streamlit
- React 或 Vue
- 聊天界面
- 会话列表
- 消息流式输出

最低要求：

- 能做一个最小聊天页面
- 能显示历史会话
- 能调用后端 `/chat`

## 阶段 10：部署和工程化

目标：让应用可以在服务器上运行。

重点内容：

- Docker
- 环境变量
- 生产启动命令
- 反向代理
- 日志文件
- 健康检查

最低要求：

- 能用 Docker 启动服务
- 能配置生产环境变量
- 能访问线上 `/health`

## 推荐学习节奏

每天 2 到 3 小时：

- 30 分钟看讲解
- 90 分钟跑代码和改代码
- 30 分钟复盘

每一节都按这个顺序：

1. 跑通
2. 看懂输入、处理、输出
3. 改一个小功能
4. 自己重写关键部分

## 当前最重要的顺序

先按这个顺序走：

1. Python 文本处理
2. HTTP 和 JSON
3. FastAPI 基础接口
4. FastAPI 常见接口类型
5. DeepSeek 聊天接口
6. 带历史记录的聊天接口
7. SQLAlchemy 和 PostgreSQL
8. RAG 文档问答

不要一开始就学 Agent 框架。先把 Web API、数据库、模型调用、历史记录这四个底座打稳。
